进程管理

进程

在系统中，每一段执行程序就成为一个进程，系统对进程进行id编号。

对于 CentOS6，管理进程的方式为 init ；

对于 CentOS7，管理进程的方式为 systemd ；

进程属性

  • 可以使用 ps 命令通过指定参数来进行查看。

    • 常用参数：

      • a

        显示当前终端关联的所有进程

      • u

        基于用户的格式显示

      • x

        显示所有进程，不以终端来区分

    • 输出参数

      • VSZ

        程序所占虚拟内存大小，单位 kB

      • RSS

        程序所占物理内存大小，单位 kB

      • TIME

        所用 CPU 时间

    • 常见用法

      • ps -aux | more

        进程过多可以使用此命令讲进程数据输出到 more

        使用 BSD 格式进行显示

      • ps -ef

        e：显示所有进程

        f：显示完整格式输出

        使用标准格式进行显示

  • 进程 ID（PID）、父进程 ID（PPID）、启动进程用户 UID、归属组织 GID、进程的状态、进程优先级、进程连接的终端名称、进程资源占用

    • 对于 GID 的输出参数

      • R

        进程正在运行中

      • S

        现在进程正在睡眠，可以通过信号量来唤醒

      • T

        目前进程正在侦测或者停止

      • Z

        Zombie僵尸进程（进程假死；进程结束，但是父进程无法停止它）

      • D

        不可终止状态

      • 特殊状态：

        • <

          表示进程具有高优先级

        • N

          表示进程具有高优先级

        • L

          进程有页面文件锁定在内存中

        • s（小写）

          表示控制进程

        • I

          表示多线程

        • +

          表示当前进程运行在前台

        • &

          表示当前进程运行在后台

注1：常用组合键
  • Ctrl+C 终止进程
  • Ctrl+Z 挂起进程
  • Ctrl+D 注销进程
  • Ctrl+/ 退出进程
注2：进程与线程

线程是进程中的一个任务，负责当前进程中程序的执行。一个进程至少有一个线程，一个进程也可以存在多个线程，多个线程可共享数据。线程共享进程的堆与方法区资源，但是每个线程有自己的程序计数器与栈。

查看系统负载

  • uptime

  • top

    动态监控命令

  • pstree

    • -u 显示进程用户
    • -t 显示进程编号
终止进程

可以通过信号/信号量来操作，或者使用命令 kill、killall、pkill。kill 后面跟 PID，若不好用，可以在后面加上信号量 -9（9 表示最高权限，强制进程结束）。如果想批量终止进程可以使用 killall 命令，例如候命跟 nginx，命令会杀死同名进程，此命令同样可加信号量以加强权限。pkill 与 kill 命令类似，但是它会杀死指定名字的所有命令。
